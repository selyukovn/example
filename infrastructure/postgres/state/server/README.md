
Для `postgres` монтируемая папка данных (`PGDATA`) на момент инициализации должна быть абсолютно пустой,
т.е. не содержать даже `.gitignore` -- соответственно, ее никак не сохранить в `git`.

Возможный костыль -- перемещать файл `.gitignore` перед запуском инициализации и возвращать его на место после нее --
требует переопределения entrypoint и добавления "возвращателя" в конец списка скриптов инициализации,
что вносит в процесс сборки образа не то чтобы очень относящиеся к нему шаги
и кажется большим из зол по сравнению с другими костылями.

Поэтому в папке `data` будет создана директория `internal` (см. [/tools.sh](../../../../tools.sh)),
которую `postgres` будет использовать для хранения данных
(cм. [/docker-compose._app_infrastructure.yml](../../../../docker-compose._app_infrastructure.yml)).

Наличие такой папки может показаться как минимум лишним уровнем вложенности, который захочется устранить --
поэтому и существует этот файл с разъяснением.
