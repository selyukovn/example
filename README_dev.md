## IDE

### Подружить автокомплит GoLand и Go в контейнере

Не получится! ...в чистом виде, но можно костылем сложить все в папку проекта.
Суть костыля -- использовать `go.mod` для скачивания зависимостей
и копирования SDK и установленных пакетов в вольюм, смонтированный под настройки IDE.

1. Создаем подпапки `sdk` и `path` в `.dev/ide-go` (в `.idea` не работает -- GoLand не видит зависимости).
2. В корне создаем `docker-compose.override.yml` и добавляем `.dev/ide-go/docker-compose.yml` в секцию `include`.
3. Выполняем `docker compose -f docker-compose.override.yml up --build` и останавливаем после вывода "Готово!".
4. В настройках GoLand ссылаем `GOROOT` на папку `.dev/ide-go/sdk`
   и `Project GOPATH` (глобальный убираем) на `.dev/ide-go/path`,
   снимаем галочку `Use GOPATH that's defined in system environment`.
5. Ждем, пока IDE все проиндексирует и подтянет дополнительные зависимости.

**Примечание**: увы, автоматическое отслеживание на данный момент недоступно,
поскольку `watch` не предназначен для наблюдения за одними и теми же файлами из разных сервисов.
Подробности см. в `.dev/ide-go/docker-compose.yml`.
Поэтому для обновления набора пакетов, доступных GoLand-автокомплиту, необходимо вручную перезапускать контейнер.
К счастью, случается это редко и требует одного единственного клика по кнопке перезапуска.

### Подружить GoLand-Debug (Delve) с контейнером

1. Создаем `docker-compose.override.yml`
2. Например, для `admin-auth-grpc` указываем `build`:`target` = `debug` и пробрасываем свой порт на `9999` порт контейнера
3. В `GoLand` добавляем конфигурацию на основе `Go Remote`,
   указывая свой проброшенный порт и отмечая `Leave it running` вариант работы контейнера при отключении.
4. В этом режиме, чтобы сервис работал, после каждого (пере-)запуска контейнера нужно запускать и GoLand-дебаггер,
   т.к. в контейнере go-код не будет выполняться, пока к серверу Delve не подключится клиент (GoLand).

**Примечание**: ожидается, что остальные go-контейнеры как и приведенный в примере
также будут открывать `9999`-й порт для дебаггера
и предоставлять специальный вариант сборки, указываемый через `build`:`target` = `debug`.

### БД в IDE

1. Создаем `docker-compose.override.yml`
2. Переопределяем нужные сервисы-бд, пробрасывая локальные порты.
   Например,
   ```
   infrastructure-mysql-replica:
      ports:
         127.0.0.1:3307:3306
   ```

## Запуск

```
. tools.sh up dev --no-migrate
```

**Примечание**: при изменении кода docker-compose перестраивает образы, но не удаляет прежние версии.
При длительной работе такие образы накапливаются и быстро съедают место на диске.
Скрипт запуска содержит автоматический `"сборщик мусора"`, который запускается параллельно `docker compose up`.
Иногда в результате гонки(?) может быть удален какой-то компонент (например, какая-то сеть), из-за чего запуск провалится.
В таком случае нужно запуститься еще раз -- пока что это терпимо.

## Инструменты

### Запуск

Скрипты запуска собраны в [`tools.sh`](tools.sh).

Команда вызова справки:
```
. tools.sh help
```

### Go

Специфические инструменты для разработки (например, go get и т.д. для go-сервисов)
уложены в специальные контейнеры `dev-tools-go-{NAME}` -- для каждого сервиса, в котором они востребованы.
Для использования нужно:

1. Создать `docker-compose.override.yml` и добавить `.dev/tools/go/docker-compose.yml` в секцию `include`.
2. После запуска подключиться к соответствующему контейнеру через IDE или вручную через терминал.

Доступные инструменты см. в [`.dev/tools/go/docker-compose.yml`](.dev/tools/go/docker-compose.yml).

### Kafka UI

https://github.com/kafbat/kafka-ui

1. Создать `docker-compose.override.yml`.
2. Расширить сервис `dev-tools-kafka-ui` из [`.dev/tools/kafka-ui/docker-compose.yml`](.dev/tools/kafka-ui/docker-compose.yml),
   пробросив порт на `8080`-порт контейнера. Например,
   ```
   dev-tools-kafka-ui:
      extends:
         file: .dev/tools/kafka-ui/Dockerfile
         service: dev-tools-kafka-ui
      ports:
         - 127.0.0.1:8003:8080
   ```

### Debezium UI

https://debezium.io/documentation//reference/stable/operations/debezium-ui.html

1. Создать `docker-compose.override.yml`. 
2. Расширить сервис `dev-tools-debezium-ui` из [`.dev/tools/debezium-ui/docker-compose.yml`](.dev/tools/debezium-ui/docker-compose.yml),
   пробросив порт на `8080`-порт контейнера. Например, 
   ```
   dev-tools-debezium-ui:
      extends:
         file: .dev/tools/debezium-ui/Dockerfile
         service: dev-tools-debezium-ui
      ports:
         - 127.0.0.1:8003:8080
   ```
